oracle权限分为两种

系统权限(大致有140来种)
	--用户对数据库的相关权限,例如:建库,建索引,修改密码...
	create session 
		*  登陆数据库的[②]角色

对象权限
	--用户对其他用户的[①数据对象]访问/操作的权限
	select	
	insert
	update
	delete
	all	表示上面所有
	create index... ...


[①]数据对象
	--用户创建的表,视图,存储过程... ....之类的东西
[②]角色
	因为oracle的权限实在是太多,挨个赋予太累,所以后了角色这一个概念,有那么点批量授权的感觉
	--connect	
	*  包含了7种权限

角色又要分两种
自定义角色
	--根据用户自己的创建
预定义角色
	--在安装数据库的时候已经配置好,没有创建便存在的
	connect	 登陆数据库的角色
	dba	 数据库管理员角色(别轻易授权,这个很叼,这个已经包含了connect的权限)
	resource 这个角色可以在任何一个表空间建表(也别轻易授权)
